VIGIL — Feature List
====================

CAPACITY
  - 4 default workspaces (up to 8)
  - 50 rooms per workspace (200 rooms default)
  - 4 devices per room (800 devices default)
  - 16-worker thread pool for parallel polling

AUTHENTICATION & SECURITY
  1.  Optional PIN protection (4+ characters)
  2.  PIN change while authenticated
  3.  PIN disable with current PIN confirmation
  4.  8-hour expiring session tokens via HTTP-only cookies
  5.  Brute force protection: 5 failed attempts lock IP for 60 seconds
  6.  PBKDF2-SHA256 PIN hashing with auto-migration from legacy SHA256
  7.  All sessions destroyed on PIN change
  8.  Auto-generated HTTPS self-signed certificate (10-year validity, RSA 2048)
  9.  Windows Root CA trust installation on first launch
 10.  Chrome certificate bypass workaround support
 11.  Security headers: X-Content-Type-Options, X-Frame-Options, Referrer-Policy, CSP, HSTS

MONITORING INFRASTRUCTURE
 12.  Continuous background device polling
 13.  Per-room poll intervals (5–120 minutes)
 14.  10-second cooldown on manual poll triggers
 15.  Parallel room polling via 8-worker thread pool
 16.  ICMP Ping check (platform-specific)
 17.  UDP port probe with ConnectionRefusedError handling
 18.  SSH port check with banner capture
 19.  HTTP/HTTPS HEAD request probe (self-signed SSL support)
 20.  Per-device timeout control (0.5–30 seconds)
 21.  Four status states: online (green), offline (red), pending (blue), degraded (amber)
 22.  Millisecond-precision latency measurement
 23.  Continuous uptime tracking (online_since timestamp)
 24.  Rolling 10-sample average latency
 25.  Status persistence across restarts via status.json
 26.  Graceful poll engine shutdown with status flush

WORKSPACE MANAGEMENT
 27.  Four default workspaces (I, II, III, IIII) — up to 8 available
 28.  Workspace rename
 29.  Independent room/device lists per workspace
 30.  Pop-out workspace in separate browser window
 31.  Quick-switch between workspaces via output buttons
 32.  URL-locked workspace view (/w/<workspace-id>)

ROOMS (MONITORING GROUPS)
 33.  Room creation (max 50 per workspace)
 34.  Room editing (name and poll interval)
 35.  Soft-delete with 24-hour recovery window
 36.  Drag-and-drop room reordering with API persistence
 37.  Live poll status display (polling now, last polled, next due)
 38.  Recently Removed list for recovering soft-deleted rooms
 39.  Permanent deletion after recovery period
 40.  Restored rooms auto-placed in original or first available workspace
